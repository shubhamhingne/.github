# Support

Thanks for using these projects. Here is how to get help.

## Before asking

1. Read the project's `README.md` and `docs/`.
2. Search existing [issues](../../issues) and [discussions](../../discussions) — your
   question may already be answered.

## How to get help

| Need | Where |
|---|---|
| A bug | Open a **Bug report** issue (use the form) |
| A feature idea | Open a **Feature request** issue |
| A usage question | Start a **Discussion** (Q&A) |
| A security issue | See [SECURITY.md](SECURITY.md) — do not open a public issue |

## What makes a good request

- A clear title and description.
- Versions, environment, and steps to reproduce.
- What you expected versus what happened.
- A minimal reproducible example where possible.

## Response expectations

These are personal open-source projects maintained on a best-effort basis. Issues are
triaged regularly; thoughtful, well-scoped reports are prioritized.
